# Recording と CARET

## 導入

Recording には少なくとも 2 つのサブステップが含まれています。ソースコードからターゲットアプリケーションを構築し、ターゲットアプリケーションを実行します。実行後、トレースポイントによってサンプリングされたイベント データを含むトレース データが表示されます。

recording で障害が発生すると、ユーザーはパフォーマンスを正しく分析できなくなるため、3 番目のステップとしてトレース データの検証が必要になります。ターゲット アプリケーションが大きい場合、または前の手順で障害が発生した場合、何らかの理由でトレース データが正しく記録されません。CARET は、ステップのトレース データを検証する機能を提供します。

CARET は、recording 中に特定のノードまたはトピックを除外するトレース フィルターをサポートしています。これはオプションですが、大規模なアプリケーションの分析には強く推奨されます。

## CARET を使用してトレース データを記録する手順

- CARETで対象アプリケーションをビルドし、確認します([See details](./build_check.md))
- (オプション) トレースフィルターの構成 ([See details](./trace_filtering.md))
- アプリケーションを記録します ([See details](./recording.md))
- トレースデータの検証([See details](./validating.md))

# TILDE、デッドライン超過を検出するフレームワーク

CARET は、期限超過を検出するフレームワーク ツールである TILDE と連携できます。
TILDE を使用すると、ROS/DDS レイヤーからは追跡できないユーザー アプリケーション内のイベントを CARET で追跡できます。

<prettier-ignore-start>
!!! Notice
        TILDEは現在開発中です。
<prettier-ignore-end>

ROSノード上ではユーザーが任意の処理を実装できるため、ノード内パスの定義が難しい場合があります。

そのような困難なケースの例を以下に挙げます。

- サブスクライブとパブリッシュの間のメッセージ バッファリングのケース
- メッセージフィルターを使用する場合

レイテンシはアプリケーション層でのみ観察できますが、CARET はイベントを観察できません。
このような場合、ユーザー コードでのデータ消費を考慮してノード内パスを定義する必要があります。ただし、CARET はユーザー コード上のイベントを監視しません。

この制約に対処するために、CARET は、TILDE がアプリケーション層のイベントを収集している間に、TILDE によって追加されたトレースポイントからのイベントを利用できます。
単一メッセージごとのメッセージ消費に注釈が付けられるため、特定のバッファリングされたメッセージを消費するコールバック関数の実行を追跡できます。
TILDE は、バッファされたメッセージの消費を追跡する機能 CARET を提供します。
